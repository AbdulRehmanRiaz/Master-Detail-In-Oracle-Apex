# Master-Detail-In-Oracle-Apex
Retail sales order system in Oracle APEX using collections and master-detail architecture, demonstrating temporary data handling, audit tracking, and final persistence to database.

Retail Sales Order System (APEX Collection + Master-Detail)

This project demonstrates a complete implementation of a Retail Order Entry System using:
Oracle APEX
APEX Collections (temporary data handling)
Master-Detail database design

Project Overview
The system allows users to:
Create a sales order (Master)
Add multiple products (Detail) using APEX Collection
Perform update/delete operations before saving
Save all data into database tables
View final saved records

Concepts Covered
APEX Collections (ADD_MEMBER, UPDATE_MEMBER, DELETE_MEMBER)
Master-Detail Architecture
Session-based data handling
Dynamic calculations
Audit columns (Created/Updated tracking)
Interactive Reports

Step 1: Database Setup

🔹 Master Table,

CREATE TABLE sales_orders (
    order_id      NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    customer_id   NUMBER NOT NULL,
    order_date    DATE DEFAULT SYSDATE,
    total_amount  NUMBER(12,2),
    created_by    VARCHAR2(100),
    created_on    DATE,
    updated_by    VARCHAR2(100),
    updated_on    DATE
);

🔹 Detail Table

CREATE TABLE sales_order_items (
    item_id        NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    order_id       NUMBER NOT NULL,
    product_id     NUMBER,
    category       VARCHAR2(100),
    brand          VARCHAR2(100),
    origin         VARCHAR2(100),
    color          VARCHAR2(50),
    stock_qty      NUMBER,
    quantity       NUMBER,
    rate           NUMBER(10,2),
    price          NUMBER(12,2),
    created_by     VARCHAR2(100),
    created_on     DATE,
    updated_by     VARCHAR2(100),
    updated_on     DATE,
    CONSTRAINT fk_order
    FOREIGN KEY (order_id)
    REFERENCES sales_orders(order_id)
);

6️⃣ Save to Database (Master + Detail)

<img src="images/master%20detail%20save%20to%20table.png" width="700"/> <br>


SELECT 
    so.order_id,
    so.order_date,
    c.full_name,
    soi.item_id,
    p.product_name,
    pc.category_name,
    soi.brand,
    soi.origin,
    soi.color,
    soi.stock_qty,
    soi.quantity,
    soi.rate,
    soi.price,
    so.total_amount,
    so.created_by,
    so.created_on,
    so.updated_by,
    so.updated_on
FROM sales_orders so
JOIN sales_order_items soi ON so.order_id = soi.order_id
join PRODUCT_CATEGORIES pc on pc.CATEGORY_ID=soi.CATEGORY
join PRODUCTS_ p on p.product_id=soi.product_id
join CUSTOMERS c on c.customer_id=so.customer_id
ORDER BY so.order_id DESC, soi.item_id;
