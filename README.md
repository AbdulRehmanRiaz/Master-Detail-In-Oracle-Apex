# Master-Detail-In-Oracle-Apex
Retail sales order system in Oracle APEX using collections and master-detail architecture, demonstrating temporary data handling, audit tracking, and final persistence to database.

Retail Sales Order System (APEX Collection + Master-Detail)

This project demonstrates a complete implementation of a Retail Order Entry System using:
Oracle APEX
APEX Collections (temporary data handling)
Master-Detail database design

Project Overview
The system allows users to:
Create a sales order (Master)
Add multiple products (Detail) using APEX Collection
Perform update/delete operations before saving
Save all data into database tables
View final saved records

Concepts Covered
APEX Collections (ADD_MEMBER, UPDATE_MEMBER, DELETE_MEMBER)
Master-Detail Architecture
Session-based data handling
Dynamic calculations
Audit columns (Created/Updated tracking)
Interactive Reports

Step 1: Database Setup

🔹 Master Table,

CREATE TABLE sales_orders (
    order_id      NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    customer_id   NUMBER NOT NULL,
    order_date    DATE DEFAULT SYSDATE,
    total_amount  NUMBER(12,2),
    created_by    VARCHAR2(100),
    created_on    DATE,
    updated_by    VARCHAR2(100),
    updated_on    DATE
);

🔹 Detail Table

CREATE TABLE sales_order_items (
    item_id        NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    order_id       NUMBER NOT NULL,

    product_id     NUMBER,
    category       VARCHAR2(100),
    brand          VARCHAR2(100),
    origin         VARCHAR2(100),
    color          VARCHAR2(50),

    stock_qty      NUMBER,
    quantity       NUMBER,
    rate           NUMBER(10,2),
    price          NUMBER(12,2),

    created_by     VARCHAR2(100),
    created_on     DATE,
    updated_by     VARCHAR2(100),
    updated_on     DATE,

    CONSTRAINT fk_order
    FOREIGN KEY (order_id)
    REFERENCES sales_orders(order_id)
);

3️⃣ Add to Collection

Data is temporarily stored using APEX Collection:

BEGIN
    APEX_COLLECTION.ADD_MEMBER(
        p_collection_name => 'ORDER_CART',
        p_c001 => :P1_PRODUCT_ID,
        p_c002 => :P1_CATEGORY,
        p_c003 => :P1_BRAND,
        p_c004 => :P1_ORIGIN,
        p_c005 => :P1_COLOR,
        p_c006 => :P1_STOCK_QTY,
        p_c007 => :P1_QUANTITY,
        p_c008 => :P1_RATE,
        p_c009 => :P1_PRICE
    );
END;


4️⃣ Collection Report (Cart View)

Displays temporary data before saving:

6️⃣ Save to Database (Master + Detail)

DECLARE
    v_order_id NUMBER;
BEGIN
    INSERT INTO sales_orders (
        customer_id, order_date, total_amount,
        created_by, created_on
    )
    VALUES (
        :P1_CUSTOMER, SYSDATE, 0,
        :APP_USER, SYSDATE
    )
    RETURNING order_id INTO v_order_id;

    FOR rec IN (
        SELECT * FROM apex_collections
        WHERE collection_name = 'ORDER_CART'
    ) LOOP

        INSERT INTO sales_order_items (
            order_id, product_id, category, brand, origin, color,
            stock_qty, quantity, rate, price,
            created_by, created_on
        )
        VALUES (
            v_order_id,
            rec.c001, rec.c002, rec.c003, rec.c004, rec.c005,
            rec.c006, rec.c007, rec.c008, rec.c009,
            :APP_USER, SYSDATE
        );

    END LOOP;

    UPDATE sales_orders
    SET total_amount = (
        SELECT SUM(c009)
        FROM apex_collections
        WHERE collection_name = 'ORDER_CART'
    )
    WHERE order_id = v_order_id;

    APEX_COLLECTION.DELETE_COLLECTION('ORDER_CART');
END;

7️⃣ Final Report (Saved Data)
SELECT 
    so.order_id,
    so.order_date,
    soi.product_id,
    soi.category,
    soi.quantity,
    soi.price,
    so.total_amount
FROM sales_orders so
JOIN sales_order_items soi
    ON so.order_id = soi.order_id
ORDER BY so.order_id DESC;
